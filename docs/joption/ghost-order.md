# 什麼是幽靈單

## 第一章：幽靈單的現象

::: info 定義
**幽靈單**指的是在委託紀錄中，「下單者」欄位顯示**空白**的委託單。這代表該筆委託**不是由目前這個程式所下**，而是來自其他來源。
:::

## 如何判斷幽靈單？

觀察委託回報中的「**下單者**」欄位：

:::tabs
== 正常單
**下單者欄位有名稱**（如「元化信選平衡程式」）

✅ 表示此單是本程式所下，屬於正常情況。

== 幽靈單
**下單者欄位是空白**

⚠️ 表示此單**不是本程式所下**，需要立即排查原因。
:::

## 最常見原因：重複開啟程式

::: danger 警告
當發現下單者是空白時，要立即確認是否**重複開啟程式**！
兩個程式同時運作會互相不知道對方的部位，造成亂下單的情況。
:::

| 情境 | 程式一 看到的下單者 | 程式二 看到的下單者 |
|------|------|------|
| 程式一下的單 | 顯示程式名稱（正常） | **空白（幽靈單）** |
| 程式二下的單 | **空白（幽靈單）** | 顯示程式名稱（正常） |

## 處理步驟

::: tip 發現幽靈單時，請依序操作
1. **停止所有程式**，先不要讓程式繼續下單
2. **確認目前部位**，手動查詢交易所的實際持倉
3. **只保留一個程式**，關閉其他重複開啟的程式
4. **重新啟動**，確認部位正確後再讓程式繼續運作
:::

## 常見 Q&A

:::tabs
== Q：幽靈單會被平倉嗎？
**A：不會自動平倉。**

幽靈單只是「顯示」問題，委託單本身仍在交易所正常掛著。程式重啟後會重新讀取目前所有部位。

== Q：幽靈單會造成損失嗎？
**A：幽靈單本身不會，但兩程式同時運作可能造成超量下單。**

例如：兩個程式各自認為需要建立部位，就會下兩倍的量，超出原本的風控設定。

== Q：如何避免幽靈單？
**A：確保同一帳號只開一個程式。**

- 每次啟動前，先確認沒有其他視窗開著
- 若要換電腦操作，先關閉原本電腦的程式
- 定期檢查委託紀錄的「下單者」欄位
:::
