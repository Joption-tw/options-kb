# 什麼是幽靈單

## 第一章：幽靈單的現象

幽靈單指的是在委託紀錄中，**「下單者」欄位顯示空白**的委託單。

這代表該筆委託**不是由目前這個程式所下**，而是來自其他來源。

## 如何判斷幽靈單

在委託回報的欄位中，觀察「下單者」欄位：

| 狀況 | 說明 |
|------|------|
| 下單者有名稱（如「元化信選平衡程式」） | 正常，是本程式所下的單 |
| 下單者是**空白** | 幽靈單，非本程式所下 |

## 發現幽靈單時怎麼辦？

> ⚠️ **當發現下單者是空白時，表示不是這一個程式下的單，要留意是不是重複開啟程式？避免兩程式亂下單情況。**

### 最常見原因：重複開啟程式

同一台電腦或不同電腦，同時開啟了兩個以上的 joption 程式，導致：

- 程式一 下的單，在程式二 的委託紀錄中顯示下單者空白（幽靈單）
- 兩個程式互相不知道對方的部位，可能造成**超量下單或方向錯誤**

### 處理步驟

1. 立即確認是否有**重複開啟**程式的情況
2. 只保留一個程式運作，關閉其餘的
3. 手動確認目前部位是否正確，必要時手動調整

## 示意圖說明

- **程式一**：委託紀錄中的下單者欄位為空白 → 幽靈單
- **程式二**：委託紀錄中的下單者欄位顯示程式名稱 → 正常單
